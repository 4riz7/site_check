# 🔍 XSS Scanner

Утилита на Python для автоматизированной проверки веб-форм на уязвимость к XSS (Cross-Site Scripting). Скрипт обходит указанные страницы, находит формы ввода, подставляет тестовый JavaScript-код и проверяет, выполняется ли он в ответе сервера.

> ⚠️ **Только для собственных ресурсов.** Инструмент предназначен для проверки безопасности сайтов, которыми вы владеете или на тестирование которых у вас есть явное разрешение. Сканирование чужих ресурсов без согласия владельца может нарушать законодательство.

## 🎯 Как это работает

1. Скрипт читает список адресов из `site.txt` (по одному URL на строку)
2. Для каждого сайта загружает HTML и находит все `<form>`
3. В текстовые поля форм подставляет тестовый payload `<Script>alert('XSS')</scripT>`
4. Отправляет форму (GET или POST — в зависимости от метода формы) и проверяет, встречается ли payload в ответе сервера в незаэкранированном виде
5. Если да — считает форму уязвимой, выводит детали в консоль и логирует URL в `goods.txt`

## ✨ Возможности

- Проверка всех форм на странице, а не только одной
- Поддержка и GET, и POST форм
- Пакетная проверка списка сайтов из файла
- Логирование уязвимых адресов в отдельный файл для дальнейшего разбора
- Маскировка запросов через реалистичный `User-Agent`
- Ограничение времени на проверку одного сайта и таймаут ответа, чтобы зависший сайт не останавливал весь прогон
- Обход ошибок отдельных сайтов без прерывания всего сканирования

## 🛠 Технологии

- **Python 3**
- [`requests`](https://pypi.org/project/requests/) — HTTP-запросы
- [`beautifulsoup4`](https://pypi.org/project/beautifulsoup4/) — парсинг HTML и поиск форм
- [`colorama`](https://pypi.org/project/colorama/) — цветной вывод результатов в консоли

## 🚀 Установка и запуск

```bash
pip install requests beautifulsoup4 colorama
```

Создайте файл `site.txt` в корне проекта и укажите в нём адреса своих сайтов, один на строку:

```
https://example-mysite.com
https://another-mysite.com
```

Запустите сканирование:

```bash
python main.py
```

Уязвимые адреса (если найдутся) появятся в консоли и в файле `goods.txt`.

## ⚙️ Настройки

В `main.py` можно поменять два параметра под свои нужды:

| Параметр | По умолчанию | Назначение |
|---|---|---|
| `stop_time` | 180 сек | максимальное время проверки одного сайта |
| `timeout` | 20 сек | таймаут ожидания ответа от сервера |
